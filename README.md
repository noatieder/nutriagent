## סקירה כללית

**NutriAgent** היא מערכת בינה מלאכותית לתמיכה תזונתית המשלבת:

- **Finite State Machine (FSM)** לאיסוף נתונים סדרתי קפדני
- **חישובים פיזיולוגיים** — BMI פדיאטרי, BMR, יעד קלורי דינמי
- **מסד נתונים מוטמע** מבוסס USDA National Nutrient Database
- **K-Means Clustering** ל-4 אשכולות מאקרו-נוטריאנטים
- **DBSCAN Guardrails** לחסימת פריטים חריגים אוטומטית
- **Cosine Similarity Engine** להחלפות ארוחות חכמות
- **Claude AI** (claude-sonnet-4) ליצירת תוכניות ארוחות מותאמות אישית
- **ממשק עברית מלא** עם נטייה מגדרית דינמית (זכר/נקבה)

### FSM — 12 מצבים סדרתיים

```
BOOT → GREETING → AGE → GENDER → WEIGHT → HEIGHT
  → [BMI/BMR TRIGGER] → ACTIVITY → ALLERGIES
  → DISLIKES → RESTRICTIONS → SUMMARY
  → [עריכה/המשך] → GENERATING → MEAL_PLAN → FOLLOWUP
```

## התקנה והרצה

### דרישות מקדימות
- דפדפן מודרני (Chrome 90+, Firefox 88+, Safari 14+, Edge 90+)
- מפתח API תקין של Anthropic (מתחיל ב-`sk-`)
- חיבור אינטרנט

### הרצה מקומית

```bash
# 1. שיבוט הריפו
git clone https://github.com/noatieder/nutriagent.git
cd nutriagent

# 2. פתיחה בדפדפן — ללא צורך בשרת!
open index.html
# או גרור את index.html לדפדפן
```

### הגדרת מפתח API
1. פתח את האפליקציה בדפדפן
2. יופיע מסך הגדרת מפתח API
3. הזן מפתח Anthropic תקין (`sk-ant-...`)
4. לחץ "הפעל את NutriAgent"

> 🔒 המפתח נשמר ב-`localStorage` בלבד ולעולם אינו עובר לשרת כלשהו מלבד שרתי Anthropic.

## זרימת המערכת

### שלב 1 — איסוף נתונים (8 שדות חובה)
| שדה | ולידציה |
|-----|---------|
| גיל | מספר שלם, טווח 4–18 |
| מין | זכר / נקבה |
| משקל | ק"ג, 10–200 |
| גובה | ס"מ, 80–220 |
| רמת פעילות | נמוכה / בינונית / גבוהה |
| אלרגיות | רשימה חופשית / אין |
| דחיות | רשימה חופשית / אין |
| הגבלות | רשימה חופשית / אין |

### שלב 2 — חישובים פיזיולוגיים
```
BMI = משקל / (גובה_במטר)²
BMR_בסיס = לפי גיל + מין (טבלה פדיאטרית)
יעד_קלורי = (BMR_בסיס + דלתא_פעילות) × מכפיל_BMI
```

| קטגוריית BMI | מכפיל קלורי |
|-------------|------------|
| תת משקל | ×1.175 (+17.5%) |
| משקל תקין | ×1.000 |
| עודף משקל | ×0.875 (-12.5%) |
| השמנת יתר | ×0.825 (-17.5%) |

### שלב 3 — יצירת תוכנית
- בניית System Prompt קליני מלא
- קריאת Claude API עם JSON schema מחייב
- סריקת DBSCAN -1 על התגובה
- ולידציית ציות לאלרגיות/הגבלות
- retry אוטומטי עד 3 ניסיונות

## מודלים של Data Science

### K-Means — 4 אשכולות מזון

| אשכול | שם | מאפיינים | דוגמאות |
|-------|-----|---------|---------|
| 0 | נפח/הידרציה | קלוריות נמוכות, סיבים גבוהים | עלים ירוקים, פירות טריים |
| 1 | חלבון רזה | חלבון גבוה, שומן נמוך | חזה עוף, טונה, קוד |
| 2 | שומנים בריאים | שומן גבוה, אנרגיה צפופה | אבוקדו, אגוזים, טחינה |
| 3 | פחמימות מורכבות | פחמימות גבוהות, סיבים | אורז מלא, קינואה, כוסמין |

### DBSCAN — זיהוי חריגים
- **3.15%** מפריטי מסד הנתונים מסווגים כ-Cluster -1
- פריטים חסומים: אבקות חלבון תעשייתיות >80g/100g, שמנים מזוקקים, סירופי סוכר
- חסימה אוטומטית למשתמשים פרטיים — דורש אישור תזונאי מוסמך

### Cosine Similarity — מנוע החלפות
```javascript
// וקטור 6 מימדי לכל פריט מזון:
vector = [calories, protein, fat, carbs, fiber, sugar]  // per serving

// כלל אדריכלי: החלפה תוך-אשכולית בלבד!
// אשכול 1 → אשכול 1 בלבד (חזה עוף → טונה / קוד / הודו)
// מניעה מוחלטת של החלפה בין אשכולות שונים
```
